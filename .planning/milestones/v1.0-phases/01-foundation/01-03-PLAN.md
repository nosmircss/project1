1. **useWeather.ts** — Custom hook for weather data fetching with retry:
   - Accepts `location: LocationInfo | null`
   - Returns `{ weather: WeatherData | null, loading: boolean, error: string | null, refetch: () => void }`
   - When location changes (lat/lon), automatically fetches weather via `window.electronAPI.fetchWeather(lat, lon)`
   - Implements silent auto-retry 2x before surfacing error (user decision):
     ```typescript
     async function fetchWithRetry(lat: number, lon: number, maxRetries = 2): Promise<WeatherData> {
       let lastError: Error
       for (let attempt = 0; attempt <= maxRetries; attempt++) {
         try {
           return await window.electronAPI.fetchWeather(lat, lon)
         } catch (err) {
           lastError = err as Error
           if (attempt < maxRetries) {
             await new Promise(r => setTimeout(r, 1000 * (attempt + 1))) // exponential backoff
           }
         }
       }
       throw lastError!
     }
     ```
   - Uses `useEffect` triggered by `location?.lat, location?.lon`
   - Sets `loading: true` before fetch, `loading: false` after success or final failure
   - On error after retries exhausted: set `error` to a user-friendly message (not raw error text)
   - `refetch` function: clears error, sets loading true, retries the fetch
   - Per user decision on stale data: if a refresh fails but `weather` state already has data, keep displaying last successful data (don't clear it). Set `error` to show the "data may be outdated" warning.
