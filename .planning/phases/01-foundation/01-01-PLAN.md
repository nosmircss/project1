  <name>Task 1: Scaffold electron-vite project with React + TypeScript and install dependencies</name>
  <files>package.json, electron.vite.config.ts, tsconfig.json, tsconfig.node.json, tsconfig.web.json, src/main/index.ts, src/preload/index.ts, src/renderer/index.html, src/renderer/src/main.tsx, src/renderer/src/env.d.ts</files>
  <action>
Scaffold the project using electron-vite's React + TypeScript template. Since we're in an existing git repo, scaffold into a temp directory then move files:

1. Run: `npm create @quick-start/electron@latest weatherdeck-tmp -- --template react-ts`
2. Move all scaffolded files from `weatherdeck-tmp/` into the project root (current directory). Do NOT overwrite `.git/` or `.planning/`.
3. Remove the `weatherdeck-tmp/` directory.
4. Install additional dependencies:
   ```bash
   npm install zipcodes-us lucide-react electron-conf
   npm install -D tailwindcss @tailwindcss/vite
   ```
5. Update `electron.vite.config.ts` to add the Tailwind CSS v4 plugin ONLY in the renderer config:
   ```typescript
   import { defineConfig, externalizeDepsPlugin } from 'electron-vite'
   import react from '@vitejs/plugin-react'
   import tailwindcss from '@tailwindcss/vite'

   export default defineConfig({
     main: {
       plugins: [externalizeDepsPlugin()]
     },
     preload: {
       plugins: [externalizeDepsPlugin()]
     },
     renderer: {
       plugins: [react(), tailwindcss()]
     }
   })
   ```
   CRITICAL: Tailwind plugin goes ONLY in `renderer` — adding it to main or preload causes build errors (see RESEARCH.md Pitfall 1).

6. Configure `src/main/index.ts` BrowserWindow with:
   - `width: 600, height: 700` (default size per user decision)
   - `minWidth: 500, minHeight: 600` (minimum for professional layout)
   - `resizable: true`
   - `backgroundColor: '#0a0a12'` (dark bg to prevent white flash before renderer loads)
   - `contextIsolation: true, nodeIntegration: false` (security best practice)
   - Remove any default menu bar with `Menu.setApplicationMenu(null)` or set `autoHideMenuBar: true`

7. Clean up the scaffold's default demo content — remove any sample components, logos, or placeholder text from `src/renderer/src/`. Keep only the entry point `main.tsx` and a minimal `App.tsx` placeholder.

8. Update `src/renderer/index.html` to add Google Fonts for the neon theme typography:
   ```html
   <link rel="preconnect" href="https://fonts.googleapis.com">
   <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
   <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=JetBrains+Mono:wght@400;700&display=swap" rel="stylesheet">
   ```

9. Verify the scaffold builds and launches:
   ```bash
   npm run build
   ```
   The build should complete without errors. Do NOT attempt to launch the Electron app in this headless environment — just verify the build succeeds.
