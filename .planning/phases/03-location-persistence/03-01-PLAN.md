**Create `src/main/locations.ts`** — a new electron-conf singleton mirroring `settings.ts`:

1. Import `Conf` from `'electron-conf/main'` and use `import type { LocationInfo }` from `'../renderer/src/lib/types'` (type-only import, erased at compile time — safe for cross-process).
2. Export `LocationsStore` interface: `{ locations: LocationInfo[], lastActiveZip: string | null, hasLaunched: boolean }`
3. Export `locationsConf` singleton: `new Conf<LocationsStore>({ name: 'locations', defaults: { locations: [], lastActiveZip: null, hasLaunched: false } })`
4. IMPORTANT: Only ONE `Conf` instance per file — electron-conf does not support multiple instances on the same config file.

**Update `src/main/index.ts`** — add 5 IPC handlers inside `app.whenReady().then(...)`, after the existing `settings:set` handler:

1. `import { locationsConf } from './locations'` at the top (alongside existing `conf` import from `./settings`)
2. `import type { LocationInfo } from '../renderer/src/lib/types'` (type-only)
3. Add handlers following the `namespace:verb` convention:

```
ipcMain.handle('locations:get-all', () => locationsConf.get('locations'))

ipcMain.handle('locations:add', (_event, location: LocationInfo) => {
  const current = locationsConf.get('locations')
  if (current.some((l) => l.zip === location.zip)) return { error: 'duplicate' }
  locationsConf.set('locations', [...current, location])
  locationsConf.set('hasLaunched', true)
  locationsConf.set('lastActiveZip', location.zip)
  return { ok: true }
})

ipcMain.handle('locations:delete', (_event, zip: string) => {
  const current = locationsConf.get('locations')
  locationsConf.set('locations', current.filter((l) => l.zip !== zip))
})

ipcMain.handle('locations:set-active', (_event, zip: string | null) => {
  locationsConf.set('lastActiveZip', zip)
})

ipcMain.handle('locations:get-meta', () => ({
  lastActiveZip: locationsConf.get('lastActiveZip'),
  hasLaunched: locationsConf.get('hasLaunched')
}))
```

Keep all IPC registrations inside the existing `app.whenReady().then(...)` block, alongside the existing handlers. Do NOT create a separate registration function.

```typescript
import { ElectronAPI } from '@electron-toolkit/preload'

interface LocationData {
  zip: string
  city: string
  stateCode: string
  lat: number
  lon: number
  displayName: string
}

interface ElectronBridgeAPI {
  fetchWeather: (lat: number, lon: number, settings?: { temperatureUnit: string; windSpeedUnit: string }) => Promise<import('../renderer/src/lib/types').WeatherData>
  getSetting: (key: string) => Promise<unknown>
  setSetting: (key: string, value: unknown) => Promise<void>
  getLocations: () => Promise<LocationData[]>
  getLocationsMeta: () => Promise<{ lastActiveZip: string | null; hasLaunched: boolean }>
  addLocation: (location: LocationData) => Promise<{ ok?: boolean; error?: string }>
  deleteLocation: (zip: string) => Promise<void>
  setActiveLocation: (zip: string | null) => Promise<void>
}

declare global {
  interface Window {
    electron: ElectronAPI
    electronAPI: ElectronBridgeAPI
    api: unknown
  }
}
```

This gives the renderer full type-safety when calling `window.electronAPI.*`.
